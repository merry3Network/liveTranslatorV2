# セットアップガイド (まずはここから)

`command not found: npm` というエラーが出る場合、お使いのパソコンに **Node.js** がインストールされていません。
以下の手順に従って環境を整えてください。

## 1. Node.js のインストール
Mac をお使いとのことですので、以下のいずれかの方法でインストールしてください。

### 方法 A: 公式インストーラーを使う (一番簡単)
1. [Node.js 公式サイト](https://nodejs.org/ja/) にアクセスします。
2. **「推奨版 (LTS)」** と書かれたボタンをクリックしてインストーラーをダウンロードします。
3. ダウンロードしたファイルを開き、画面の指示に従ってインストールを完了させます。

### 方法 B: Homebrew を使う (エンジニア向け)
ターミナルで以下のコマンドを実行します。
```bash
brew install node
```

---

## 2. インストールの確認
ターミナルを**一度閉じて、新しく開いてください**。
その後、以下のコマンドを入力してバージョン番号が表示されれば成功です。

```bash
node -v
npm -v
```

---

## 3. プロジェクトの準備とテスト
Node.js が入ったら、**まずはプロジェクトのフォルダに移動する必要があります**。
以下のコマンドを実行してください。

```bash
cd /Users/rsk/.gemini/antigravity/scratch/livetranstestmodelv1
```

移動できたら、以下のコマンドを順番に実行してください。

### 依存関係のインストール
必要なライブラリをダウンロードします。
```bash
npm install
```

### テストの実行
作成したテストコードが正しく動くか確認します。
```bash
npm test
```

### アプリの起動（開発モード）
ローカルでアプリを動かすには、**2つのターミナル（黒い画面）** を開いて、それぞれで別のコマンドを実行する必要があります。

**ターミナル1（バックエンド用）**
サーバーを起動します。この画面は**開いたまま**にしてください。
```bash
node server.js
```
※ 「Secure Proxy Server ... running on port 8080」と表示されれば成功です。

**ターミナル2（フロントエンド用）**
新しいターミナルを開き、同じフォルダに移動してから実行します。
```bash
cd /Users/rsk/.gemini/antigravity/scratch/livetranstestmodelv1
npm run dev
```
起動後、表示される URL (例: `http://localhost:5173`) をブラウザで開いてください。
